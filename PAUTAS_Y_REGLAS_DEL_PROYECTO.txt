FILOSOFÍA PRINCIPAL DEL PROYECTO
================================================================================

1. LAS VÍCTIMAS SON EL CENTRO, NO LOS CRIMINALES
   - Nunca glorificamos a los asesinos
   - Los criminales solo se mencionan para dar contexto
   - El foco siempre está en recordar a las víctimas como PERSONAS

2. DIGNIDAD ANTE TODO
   - No usamos fotos sensacionalistas
   - No describimos detalles morbosos innecesarios
   - Tratamos cada caso con respeto

3. INCLUSIVIDAD
   - El lenguaje es inclusivo (no solo femenino ni solo masculino)
   - Incluimos víctimas de todos los géneros
   - Casos de múltiples países y culturas

REGLAS DEL BUSCADOR
================================================================================

1. BÚSQUEDA SIN TILDES
   - Cada campo tiene una versión "Search" sin tildes
   - "Alcàsser" → titleSearch: "alcasser"
   - "María" → fullNameSearch: "maria"
   - El usuario puede buscar con o sin tildes

2. BÚSQUEDA CASE-INSENSITIVE
   - "DAHMER" encuentra "Dahmer"
   - "madeleine" encuentra "Madeleine"

3. CAMPOS BUSCABLES
   - Título del caso
   - País y ciudad
   - Nombre de víctimas
   - Aliases de víctimas

4. FUNCIÓN DE NORMALIZACIÓN
   ```
   function normalizeForSearch(text) {
     return text
       .toLowerCase()
       .normalize('NFD')
       .replace(/[\u0300-\u036f]/g, '')  // Quita tildes
       .replace(/[^\w\s]/g, '')          // Quita caracteres especiales
       .trim()
   }
   ```

REGLAS DE ORTOGRAFÍA Y ESCRITURA
================================================================================

1. SIEMPRE USAR TILDES CORRECTAS
   - niña (NO nina)
   - año (NO ano)
   - día (NO dia)
   - había (NO habia)
   - policía (NO policia)

2. LENGUAJE INCLUSIVO
   - "las víctimas" o "quienes sufrieron" (NO "las mujeres que...")
   - "eran parte de una familia" (NO "eran hijas, madres...")
   - Incluir pronombres neutros cuando sea posible

3. TONO RESPETUOSO
   - Evitar palabras sensacionalistas
   - No dramatizar innecesariamente
   - Enfocarse en la humanidad de las víctimas

4. NOMBRES CORRECTOS
   - Verificar ortografía de nombres propios
   - Usar apellidos con tildes correctas (García, Martínez, López)

================================================================================
TIPOS DE CASOS SOPORTADOS
================================================================================

1. HOMICIDE (Homicidio)
   - Asesinatos individuales
   - Casos resueltos o sin resolver

2. DISAPPEARANCE (Desaparición)
   - Personas desaparecidas
   - Con o sin cuerpo encontrado

3. SERIAL_KILLER (Asesino en Serie)
   - Múltiples víctimas
   - Un solo perpetrador o grupo

4. COLD_CASE (Caso Frío)
   - Sin resolver por muchos años
   - Investigación estancada

5. KIDNAPPING (Secuestro)
   - Secuestros con o sin rescate
   - Con cualquier desenlace

6. OTHER (Otro)
   - Casos que no encajan en otras categorías
   - Supervivientes, casos especiales

================================================================================
ESTADOS DE VÍCTIMAS
================================================================================

1. DECEASED (Fallecida)
   - La víctima murió

2. MISSING (Desaparecida)
   - Nunca se encontró
   - Estado desconocido

3. SURVIVED (Sobrevivió)
   - Víctima que sobrevivió al ataque
   - Ejemplo: Nayoung, Bryan Hartnell

CÓMO AÑADIR UN NUEVO CASO
================================================================================

PASO 1: Crear el caso en la base de datos
```typescript
await prisma.case.create({
  data: {
    slug: 'nombre-del-caso',
    title: 'Título del Caso',
    titleSearch: normalizeForSearch('Título del Caso'),
    summary: 'Resumen corto...',
    status: 'SOLVED' o 'UNSOLVED',
    type: 'HOMICIDE', 'DISAPPEARANCE', etc.,
    year: 2024,
    country: 'País',
    countrySearch: normalizeForSearch('País'),
    city: 'Ciudad',
    citySearch: normalizeForSearch('Ciudad'),
    mainImageUrl: '/images/cases/nombre-caso.jpg',
    content: `## Historia completa en markdown...`,
    victims: {
      create: [
        {
          fullName: 'Nombre de Víctima',
          fullNameSearch: normalizeForSearch('Nombre'),
          // ... todos los campos de víctima
        }
      ]
    },
    timelineEvents: {
      create: [
        { date: new Date('2024-01-01'), title: 'Evento', description: '...', order: 1 }
      ]
    }
  }
})
```

PASO 2: Añadir datos de impacto
```typescript
await prisma.caseImpact.create({
  data: {
    caseId: caso.id,
    legislationName: 'Nombre de Ley',
    legislationYear: 2024,
    legislationDesc: 'Qué hace la ley...',
    communityImpact: 'Cómo afectó a la comunidad...',
    // ... más campos
  }
})
```

PASO 3: Añadir imagen
- Colocar imagen en /public/images/cases/
- O usar placeholder existente

PASO 4: Verificar ortografía
- Ejecutar script de corrección ortográfica
