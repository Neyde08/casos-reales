PARTE 1: ¿QUÉ ES ESTA PÁGINA?
================================================================================

Imagina que tienes un álbum de fotos, pero en vez de fotos de vacaciones,
tiene historias de personas que merecen ser recordadas. Esta página es como
ese álbum, pero en internet.

La página se llama "Casos Reales" y es especial porque:
- No nos enfocamos en los "malos" (los criminales)
- Nos enfocamos en las VÍCTIMAS (las personas que sufrieron)
- Queremos que la gente las recuerde como eran: personas con sueños y familias

================================================================================
PARTE 2: LAS HERRAMIENTAS QUE USAMOS
================================================================================

Para construir una casa necesitas martillo, clavos y madera.
Para construir una página web necesitas:

1. NEXT.JS (versión 16)
   - Es como el plano de la casa
   - Le dice a la computadora cómo organizar todo
   - Hace que la página sea rápida y bonita

2. REACT (versión 19)
   - Es como los ladrillos de LEGO
   - Cada parte de la página (botones, menús, tarjetas) es un "componente"
   - Podemos reutilizar los mismos ladrillos en diferentes lugares

3. TYPESCRIPT
   - Es como un supervisor que revisa que todo esté bien escrito
   - Si cometes un error, te avisa antes de que la página se rompa

4. TAILWIND CSS (versión 4)
   - Es como la pintura y decoración
   - Hace que la página se vea bonita con colores y espacios

5. PRISMA + SQLITE
   - Es como un archivero gigante donde guardamos toda la información
   - SQLite es el archivero
   - Prisma es el asistente que nos ayuda a buscar y guardar cosas

PARTE 4: LA BASE DE DATOS - DONDE GUARDAMOS TODO
================================================================================

Imagina que tienes fichas de colección de tus personajes favoritos.
Cada ficha tiene información organizada:

FICHA DE UN CASO:
┌─────────────────────────────────────┐
│ Título: El Crimen de Alcàsser      │
│ País: España                        │
│ Año: 1992                           │
│ Estado: RESUELTO                    │
│ Tipo: HOMICIDIO                     │
│ Resumen: Tres chicas fueron...      │
└─────────────────────────────────────┘

FICHA DE UNA VÍCTIMA:
┌─────────────────────────────────────┐
│ Nombre: Miriam García Iborra        │
│ Edad: 14 años                       │
│ Personalidad: Era alegre y...       │
│ Sueños: Quería ser veterinaria...   │
│ Familia: Sus padres la adoraban...  │
│ Lo que le gustaba: La música y...   │
└─────────────────────────────────────┘

Estas "fichas" en programación se llaman MODELOS.
Tenemos varios modelos:

- Case (Caso): La información general del crimen
- Victim (Víctima): Información de cada persona afectada
- TimelineEvent (Línea de tiempo): Qué pasó y cuándo
- Suspect (Sospechoso): Información de los acusados
- Evidence (Evidencia): Pruebas del caso
- Tribute (Tributo): Mensajes de visitantes para las víctimas
- CaseImpact (Impacto): Cómo el caso cambió leyes o la sociedad

PASO 5: MEJORAR EL BUSCADOR
─────────────────────────────────────────
PROBLEMA: Si escribías "alcasser" no encontraba "Alcàsser"
          porque la computadora veía las tildes como letras diferentes.

SOLUCIÓN: Creamos un "traductor" que quita los acentos para buscar:
          "Alcàsser" → "alcasser"
          "María" → "maria"
          "niño" → "nino"

Ahora puedes buscar sin preocuparte por tildes o mayúsculas.

PARTE 7: CÓMO FUNCIONA LA BÚSQUEDA (EJEMPLO SENCILLO)
================================================================================

Imagina que tienes una caja de crayones con nombres:
- "Rojo"
- "Azúl" (con tilde mal puesta)
- "Verde"

Si buscas "azul" (sin tilde), no encontrarías "Azúl".

SOLUCIÓN:
Creamos una "copia secreta" de cada nombre sin tildes:
- "Rojo" → copia secreta: "rojo"
- "Azúl" → copia secreta: "azul"
- "Verde" → copia secreta: "verde"

Cuando buscas "azul", la computadora busca en las copias secretas,
¡y así encuentra "Azúl" aunque escribas sin tilde!

En el código esto se ve así:

    function normalizeForSearch(text) {
      return text
        .toLowerCase()           // "HOLA" → "hola"
        .normalize('NFD')        // Separa las tildes
        .replace(/tildes/g, '') // Quita las tildes
    }

PARTE 9: CONSEJOS SI QUIERES APRENDER A PROGRAMAR
================================================================================

1. EMPIEZA PEQUEÑO
   No intentes hacer una página enorme al principio.
   Empieza con algo simple como "Hola Mundo".

2. LEE EL CÓDIGO DE OTROS
   Es como leer libros para aprender a escribir.
   Mira cómo otros programadores resuelven problemas.

3. NO TENGAS MIEDO DE LOS ERRORES
   Los errores son tus amigos. Te enseñan qué está mal.
   Hasta los programadores expertos cometen errores todo el tiempo.

4. USA HERRAMIENTAS QUE TE AYUDEN
   - VS Code (editor de código)
   - GitHub (para guardar tu código)
   - ChatGPT o Claude (para preguntar dudas)

5. PRACTICA TODOS LOS DÍAS
   Aunque sea 15 minutos. La práctica hace al maestro.

PARTE 10: GLOSARIO (PALABRAS RARAS EXPLICADAS)
================================================================================

API: Application Programming Interface
     Es como un mesero en un restaurante. Tú le pides algo (datos)
     y él va a la cocina (servidor) a buscarlo y te lo trae.

Base de datos: Un archivero digital donde guardamos información organizada.

Componente: Una pieza de LEGO del código. Un botón, un menú, una tarjeta.

Framework: Un conjunto de herramientas ya hechas que te facilitan el trabajo.
           Next.js es un framework.

Frontend: La parte de la página que VES (colores, botones, texto).

Backend: La parte de la página que NO ves (servidores, bases de datos).

JSON: Una forma de escribir datos que las computadoras entienden fácil.
      Ejemplo: { "nombre": "Juan", "edad": 10 }

Placeholder: Una imagen o texto temporal mientras no tenemos el real.

Query: Una pregunta que le hacemos a la base de datos.
       "Dame todos los casos de España" es una query.

Schema: El "mapa" que describe cómo está organizada la base de datos.

Script: Un programa pequeño que hace una tarea específica.

Slug: Una versión "amigable" de un título para usar en URLs.
      "El Crimen de Alcàsser" → "crimen-alcasser"
